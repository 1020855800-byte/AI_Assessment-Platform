# 产品需求文档：翰闻人工智能考评系统V1.0

## 1. 项目概述

### 1.1 背景

暂无

### 1.2 系统范围

本项目包含三个独立但互联的前端应用：

- 翰闻人工智能考评管理后台（系统管理端）：内部运营人员使用。
- 翰闻数据标注实训管理后台（学校/机构管理后台）：学校教务/老师使用。
- 翰闻数据标注实训平台（考生端）：考生参加考试使用（本期仅需完成登录鉴权，具体考试界面接入开源系统）。

## 2. 用户角色与核心需求

|  |  |  |  |
| --- | --- | --- | --- |
| **角色** | **所属端** | **核心痛点** | **核心任务** |
| 平台运营员 | 系统管理端 | 如何给学校开账号、续费、排障 | 创建学校账号、重置密码、设置有效期、快速切换登录 |
| 教务/老师 | 机构管理后台 | 如何发起一场线上考试并管理考生 | 创建考试计划、导入考生、控制考试过程、导出考试过程结果文件 |
| 考生 | 考生端 | 如何登录并完成考试 | 通过身份证号+密码登录，进入考试系统 |

### 端1：翰闻人工智能考评管理后台（系统管理端）

**定位**：平台运营方后台，不面向租户开放。

|  |  |  |
| --- | --- | --- |
| **模块** | **功能点** | **需求描述 (PRD Detail)** |
| **租户管理** | 创建学校账号 | 表单字段：学校/机构全称（唯一）、管理员姓名、联系电话、初始密码（系统生成或自定义）。点击“创建”后，系统自动生成唯一的学校ID。 |
| **权限控制** | 设置账号使用时间段 | **字段**：账号有效开始日期、账号有效结束日期（精确到日）。  **规则**：到期当天23:59:59后，该学校账号无法登录管理后台，且其下属考生无法登录考生端。支持续费（修改日期）。 |
| **账号维护** | 重置密码 | 支持一键重置学校管理员密码。重置后，系统强制提示“新密码已发送至管理员手机/邮箱”（演示环境可明文显示一次，生产环境需加密发送）。 |
| **运营提效** | 快捷跳转登录 | 在学校账号列表页，设置【登录此学校】按钮。点击后，系统管理端**免密**直接跳转至该学校的机构管理后台，且身份为该学校的超级管理员。用于客服或运营协助排查问题。 |

### 端2：翰闻数据标注实训管理后台（学校/机构管理后台）

**定位**：学校/机构的教学管理中枢。

#### 4.2.1 登录与安全

|  |  |  |
| --- | --- | --- |
| **模块** | **功能点** | **需求描述** |
| **登录鉴权** | 账号密码校验 | 输入账号、密码。校验失败提示“账号或密码错误”。支持记住密码。 |
| **时间段校验** | 登录时立即校验当前服务器时间是否在学校账号的有效期内。若不在，提示“您机构的账号已过期，请联系平台续费”，拦截登录。 |  |
| **个人设置** | 修改密码 | 需输入“旧密码” -> “新密码” -> “确认新密码”。规则：密码长度8-16位，必须包含字母与数字。 |

#### 4.2.2 考试计划全生命周期管理

|  |  |  |
| --- | --- | --- |
| **阶段** | **功能点** | **需求描述** |
| **创建** | 创建考试计划 | 表单包含：  1. 计划名称（如“2025年第一期标注师认证”）；  2. 考评项目（下拉框，可自定义，如“图像标注”）；  3. 等级（初级/中级/高级）；  4. **考试时间段**（开始时间 ~ 结束时间，精确到分钟）；  5. **考试时长**（单位：分钟，例如90分钟）。 |
| **准备** | 导入考生 | 支持单个新增/批量导入（下载Excel模板）。  **考生字段**：姓名、身份证号（唯一标识）、手机号。  **校验**：同身份证不可重复导入同一场考试；身份证号格式校验。 |
| **准备** | 自主创建开关 | 【是否允许考生自主创建项目】开关。   * 开启：考生登录后，可自行报名新的考试项目。 * 关闭：仅管理员导入的考生可参加（默认推荐）。 |
| **发布** | 计划编辑与发布 | 草稿状态下可任意编辑。点击【发布考试计划】后，状态变更为“待考试”，考生端可见该计划（若有权限）。  **注意**：发布后不允许修改考试时间段及时长。 |
| **执行** | 考生额外加时 | 在考试计划详情 -> 考生列表，对特定考生点击【加时】。输入加时分钟数（需大于0）。该考生端倒计时相应延长。需记录加时操作日志。 |
| **结束** | 办结 | 考试时间截止后，计划状态进入“待办结”。管理员确认无误后，点击【办结】按钮，该考试计划变为“已结束”状态，不再允许任何考生提交试卷或登录考试。办结后才能导出完整结果。 |
| **监控** | 考生行为记录 | 系统自动记录并展示：   * 登入时间（首次进入试卷时间） * 登出时间（提交试卷/系统强制交卷时间） * 登录IP（考生端请求IP） * 作答状态（未开始/考试中/已交卷/超时未交） |
| **输出** | 导出考试结果 | 点击【导出】按钮，生成Excel文件。包含：考生姓名、身份证号、手机号、考试得分（若接入的开源系统提供）、登录IP、登入登出时间、作答状态。 |

### 端3：翰闻数据标注实训平台（考生端）

**定位**：考生入口。因明确要求**接入其他开源系统**，本节仅定义鉴权与交接部分。

|  |  |  |
| --- | --- | --- |
| **模块** | **功能点** | **需求描述** |
| **考生登录** | 身份验证 | **登录凭证**：身份证号 + 密码。  **密码规则**：初始密码为身份证号后6位（若身份证含X，X大写）。  **校验逻辑**：  1. 校验身份证是否在任意“已发布”或“进行中”的考试计划考生列表中。  2. 校验当前时间是否在该考试计划的有效时间段内。  3. 校验该考生所在机构的账号是否在有效期内。  通过后颁发Token。 |
| **页面跳转** | 系统对接 | 登录成功后，请求后端获取该考生第一个待考试的考试计划URL。  前端**重定向**至该开源系统的指定地址，并在URL中携带用户的唯一标识（UserID或Token）。  **注**：开源系统的具体界面、交卷逻辑、得分计算不在本期PRD范围内。 |
| **兜底** | 登出/登入记录 | 在跳转至开源系统前，考生端前端必须向后端上报一条“登入时间、IP”记录。交卷/超时时由后端监听（需开源系统回调）或由考生端每隔5分钟发送心跳。 |

## 6. 非功能性需求（NFR）

- 性能：考试开始前后5分钟，系统需支持300 QPS的登录并发（按单校2000人同时开考设计）。
- 安全： 所有密码（包括学校管理员密码、考生密码）必须使用bcrypt加密存储。
- 禁止考生通过前端JS修改考试时长或提交时间。
- 学校管理后台需配置操作日志，记录谁在什么时间导出了考生数据。
- 可用性：考生登录失败时，提示必须明确但不暴露敏感信息（如：不提示“密码错误”而是提示“身份证号或密码不正确”；不提示“账号过期”而是提示“当前无法登录，请联系学校管理员”）。
- 可扩展性：预留“是否开启人脸识别”、“是否防切屏”等配置项，便于V2.0迭代。

## 7. 附录：异常场景处理

|  |  |
| --- | --- |
| **场景** | **系统行为** |
| 学校账号过期当天有考试 | 系统管理后台应提前3天弹窗提醒运营人员。过期后，该学校下所有考生登录时直接拦截，并给出友好提示。 |
| 考生导入时身份证重复 | 前端提示“第X行，身份证号XXXX重复，请检查”，阻止导入。 |
| 考试进行中，管理员误触“办结” | 二次确认弹窗，文字警告：“办结后将禁止所有考生提交，已有作答记录将被保存。确认办结吗？” |
| 考生在考试结束前1秒提交 | 以服务器接收请求的时间为准。若服务器时间超过截止时间+1秒，拒绝提交并提示“考试已结束”。 |
